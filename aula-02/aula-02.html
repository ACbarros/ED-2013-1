1.2 Loop-free

  • conta os comandos
  • lista tamanho n call de append é \(O(1)\) e insert(0, x) é \(O(n)\). Considere 
nums.append(1)
nums.insert(0,2)

Qual complexidade? \(O(1) + O(n+1) = O(1) + O(n) = O(n)\).

1.3 Diferença entre insert e append

Com append, quase instantâneo. Linear. 

count = 10**5
nums = []
for i in range(count):
    nums.append(i)
nums.reverse()

Com insert, demora, quadrático. Por que? 

count = 10**5
nums = []
for i in range(count):
    nums.insert(0, i)
nums

Listas em Python são vetores/arrays, não listas encadeadas.

1.4 Loops

Cases \(O(n)\): 

s = 0 
for x in seq:
    s += x

squares = [x**2 for x in seq]

Outros casos do Livro PyAlgo, "taking the asymptotics for a Spin". Ultimo caso, ainda \(O(n^2)\). Em Lisp para praticarmos! 

(defun range (start last)
  (loop :for i :from start :to last 
        :collect i))

(let* ((seq (range 1 5))
       (n (length seq))
       (s nil))
  (dolist (i (range 0 (- n 2)) (nreverse s))
    (dolist (j (range (1+ i) (- n 1)) s)
      (push (list i j) s))))
